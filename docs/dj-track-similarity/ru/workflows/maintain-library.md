# Безопасное обслуживание фонотеки v7

> Для кого: Для тех, кто поддерживает локальное музыкальное состояние в порядке.
> Задача: Обслуживать набор файлов v7, не изменяя исходное аудио.
> Тип: Сценарий

1. Сканируйте после добавления файлов. Сканирование обновляет треки и теги Core, не записывая аудио.
2. Храните Core и обязательный `*.artifacts.sqlite` вместе: они связаны `catalog_uuid`.
3. Считайте `*.evaluation.sqlite` необязательным состоянием оценки.
4. Перед применением переноса выполните предварительную проверку. Применение меняет только
   сохранённый `tracks.file_path`; файлы не перемещаются, не копируются, не удаляются и не получают
   новые теги.
5. Запускайте Audio Doctor и Audio Dedup в режиме отчёта до режимов применения с подтверждением.

При изменении идентичности SONARA не пытайтесь мигрировать v5/v6 и не смешивайте релизы. Запустите
упорядоченный и продолжаемый после сбоя сценарий `prepare-sonara-release` с резервными копиями,
затем повторно проанализируйте и пересоберите каждый классификатор, зависящий от SONARA. Это не
распределённая атомарная транзакция.

Запускайте `python scripts\optimize_database.py --db .\data\library.sqlite` только для локального
набора файлов под вашим контролем. Скрипт проверяет и копирует Core + Artifacts до обслуживания. Порт
frontend v7 отложен, поэтому используйте CLI/API, а не текущие элементы браузера.
