# Карта документации

> Для кого: Для тех, кто хочет увидеть весь проект перед выбором конкретной задачи.
> Задача: Найти нужную страницу для установки, ежедневной работы, обслуживания или проверки контракта.
> Тип: Объяснение

DJ Track Similarity помогает искать музыку в локальной коллекции, когда папок, тегов и памяти уже
недостаточно. Отправной точкой может быть знакомый трек, описание нужного звучания или черновая идея
сета. Приложение превращает её в короткий список, который стоит прослушать.

Приложение не решает, какие треки «хорошие», и не гарантирует, что любая найденная пара сведётся
безупречно. Оно сокращает область поиска, объясняет появление кандидата и оставляет музыкальное
решение диджею.

## Что можно получить

| Ваша ситуация | С чего начать | Результат |
| --- | --- | --- |
| Помните один подходящий трек | Поиск по референсному треку (seed) в MERT или SONARA | Ранжированный список близких кандидатов для прослушивания |
| Можете описать звук, но не назвать трек | Текстовый поиск CLAP | Короткий список, подобранный по описанию звучания |
| Есть несколько опорных точек для сета | Smart Set Builder | Редактируемый черновик порядка с направлением энергии и темпа |
| Нужен широкий пул для последующего прослушивания | Фильтры, референсные треки или текстовый поиск | Подборка, которую можно экспортировать в CSV или M3U |
| Регулярно оцениваете треки по собственному признаку | Классификатор Rhythm Lab | Сохранённая оценка каждого трека для фильтров и мягкого влияния на SET или Hybrid |

Эта карта поможет выбрать самый короткий путь от текущей идеи к нужному результату. Точные
контракты CLI, API, хранилища и моделей находятся в справочном разделе.

## Если вы только начинаете

1. Прочитайте раздел [Установка](./getting-started/install.md).
2. Пройдите [Быстрый старт](./getting-started/quickstart.md).
3. Создайте базу по инструкции [Первая фонотека](./getting-started/first-library.md).
4. Добавьте результаты моделей с помощью страницы [Первый анализ](./getting-started/first-analysis.md).
5. Соберите первый список через [поиск по референсным трекам](./user-guide/search-with-seeds.md) или [текстовый поиск](./user-guide/text-search.md).
6. Прослушайте кандидатов, прежде чем превращать их в подборку или сет.

## Текущая команда сборки документации

Запускайте из активированного окружения после установки зависимостей Node.js:

```powershell
cd docs\dj-track-similarity
npm run check
```

`npm run check` выполняет строгую проверку Vale для `README.md` и дерева Markdown VitePress, а затем
собирает статический сайт в `site/`.
