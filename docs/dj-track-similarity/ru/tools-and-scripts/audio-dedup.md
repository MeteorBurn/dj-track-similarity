# Audio Dedup

> Для кого: Для тех, кто ищет кандидатов в дубликаты аудио.
> Задача: Безопасно создать отчёт и понимать режим применения.
> Тип: Руководство

По умолчанию Audio Dedup читает существующую библиотеку SQLite и создаёт отчёты JSON, XLSX и журнал.
Он использует сохранённый анализ и локальные пути, не сканируя неизвестные папки за пределами
выбранного корня.

## Требования

Инструменту нужны сохранённые сигналы аудио–аудио. В оценке дубликатов используются эмбеддинги MERT,
MAEST и CLAP вместе с другими безопасными проверками ядра. `min_similarity` не относится к шкале
текстового поиска CLAP.

## Работа в интерфейсе

Откройте Audio Dedup значком копирования в верхней панели.

Настройки:

- **Root**: рассматриваются только треки базы внутри этого сохранённого корня пути.
- **Path contains**: необязательные регистронезависимые фильтры пути, разделённые строками, запятыми или точками с запятой.
- **Preset**: safe, balanced или aggressive.
- **Min score**: необязательное переопределение `0..1`.
- **Min similarity**: необязательный порог содержимого аудио–аудио `0..1`.
- **Limit groups**: необязательное максимальное число групп.
- **Output dir**: каталог отчётов.

Нажмите **Start**, чтобы создать отчёт. Интерфейс покажет прогресс и по завершении откроет XLSX.

## Отчёт из CLI

```powershell
python tools\audio-dedup\audio_dedup_cli.py --db .\data\library.sqlite --root D:\Music --preset safe
```

Дополнительный пример:

```powershell
python tools\audio-dedup\audio_dedup_cli.py --db .\data\library.sqlite --root D:\Music --path-contains wav --limit-groups 50
```

## Режим применения

Применение разрушительно и требует точного подтверждения:

```text
APPLY DELETE
```

Инструмент удаляет только безопасных кандидатов внутри выбранного корня и убирает из SQLite только
те треки, чьи файлы действительно были удалены.

Не запускайте применение в обычных тестах. Сначала изучите отчёт, а для важной библиотеки сохраните
резервные копии.

## Выходные данные

По умолчанию отчёты JSON, XLSX и журнал создаются под `tools/audio-dedup/data/reports/`. Это личные
локальные артефакты.
