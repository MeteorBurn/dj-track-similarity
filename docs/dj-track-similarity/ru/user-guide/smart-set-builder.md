# Создайте черновик в Smart Set Builder

> Для кого: Для тех, кто готовит упорядоченный список для прослушивания или диджейского сета.
> Задача: Объяснить настройки SET, требования, предварительный просмотр Hybrid и границы безопасности.
> Тип: Руководство

SET помогает после того, как поиск нашёл слишком много правдоподобных треков, а вам всё ещё нужны
музыкальное направление и порядок проверки. Укажите опорные точки или разрешите приложению выбрать
их, затем задайте желаемую близость, разнообразие, форму энергии и, при необходимости, направление
темпа или классификатора.

Результат — упорядоченный черновик для прослушивания. По нему удобно искать возможный маршрут через
библиотеку и замечать пробелы, повторы или резкие скачки. Это не готовый план выступления, не
генератор cue points и не гарантия того, что соседние треки сведутся.

## Что происходит при создании

1. SET ограничивает пул треками с необходимым анализом.
2. Находит кандидатов, связанных с опорными точками несколькими аудиосигналами.
3. Балансирует кандидатов по режиму, разнообразию, энергии, темпу, исполнителям и необязательным настройкам классификаторов.
4. Возвращает упорядоченный черновик со сведениями о покрытии.
5. До нажатия **Add preview** текущий сет не изменяется.

Вкладка **SET** создаёт черновик через `/api/set-builder/generate`.

## Требования

SET использует треки с полным набором признаков. Трек подходит только при наличии:

- анализа SONARA;
- эмбеддинга MERT;
- эмбеддинга MAEST;
- аудиоэмбеддинга CLAP.

Ответ содержит общее и подходящее количество треков, а также число отсутствующих результатов MERT,
MAEST, CLAP и SONARA.

## Источник опорных точек

**Manual - selected** использует выбранные треки как путевые точки. Бэкенд проверяет полноту их
признаков и применяет защиту от повторов: не более одного трека известного исполнителя в одном
черновике.

**Auto - random start** выбирает первую точку из библиотеки с полными признаками, а затем подбирает
связанные промежуточные ориентиры. Значение **Auto anchors** ограничивается диапазоном `1..5`.

## Режим сета

- **Similar crate - close** остаётся близко к ориентирам и реже рискует разнообразием.
- **Weird adjacent - odd** допускает менее очевидный соседний материал, сохраняя связь с ориентирами.
- **Balanced set - flow** балансирует сходство, разнообразие, совместимость перехода, кривую энергии и ограничения исполнителей.
- **Discovery - wide** расширяет поиск ради новизны, но сохраняет связь кандидатов с ориентирами.

Выбирайте режим по тому, что хотите услышать. Close полезен для цельной подборки, Balanced — хороший
первый черновик, а более широкие режимы находят неочевидные связи, которым требуется внимательная
проверка на слух.

## Размер и разнообразие

- **Track limit** принимает `1..500`. Опорные и промежуточные точки входят в этот лимит.
- **Diversity** принимает `0.00..1.00`. Малое значение держится ближе к ориентирам, большое сильнее разносит кандидатов, сохраняя связь.

## Кривая энергии

- **Balanced** удерживает энергию рядом с контекстом опорных точек.
- **Warmup** начинает ниже и постепенно поднимается.
- **Peak** предпочитает высокую энергию и плотность.
- **Wave** создаёт чередование подъёма и спада.

## Настройки BPM

Режим **General BPM** по умолчанию использует BPM и тональность только как мягкие сигналы
совместимости перехода. Сначала SET берёт актуальные подписанные данные темпа SONARA. При уверенности
ниже `0.45` он проверяет кандидатов SONARA и тег BPM из Mutagen. `grid_stability` может ослабить
надёжность темпа; при низкой надёжности оценка движется к нейтральной, а не превращается в бонус
сходства или жёсткое исключение.

Для тональных переходов SET сначала предпочитает корректный тег Camelot, затем результат Camelot
SONARA, а после этого преобразует обычное имя тональности. Одинаковые, относительные и соседние
тональности получают градуированную совместимость, а не проверку простого совпадения строк. Низкая
уверенность SONARA ослабляет гармонический сигнал к нейтральному и не превращает совпадение двух
сомнительных значений в сходство само по себе.

Явные режимы траектории:

- **Low to high**: движение от меньшего BPM к большему.
- **High to low**: движение от большего BPM к меньшему.

При активной траектории:

- **BPM change** может быть slow, medium или fast;
- **Start BPM** и **Target BPM** принимают `20..300` либо остаются пустыми для автоматического выбора;
- половинный и двойной темп учитываются для совместимости перехода, но не заменяют фактическую траекторию BPM.

## Ползунки классификаторов

Опубликованные классификаторы могут добавлять предпочтение SET:

- **Preference** находится в диапазоне от `-1.00` до `1.00`;
- **Flow** может быть flat, rise или fall.

Отсутствующая оценка классификатора остаётся нейтральной. Эти элементы читают сохранённые баллы, но
не обучают модель и не декодируют аудио.

## Предварительный просмотр Hybrid

Во вкладке SET находится **Hybrid preview** — явный взвешенный просмотр по сохранённым данным MERT,
MAEST, SONARA и CLAP.

Hybrid:

- требует от одного до пяти выбранных опорных треков;
- позволяет включать и отключать каждый источник;
- принимает веса источников от `0.00` до `1.00`;
- запрашивает от `1` до `100` кандидатов на источник;
- показывает до `1..100` строк;
- применяет необязательный штраф риска перехода от `0.00` до `1.00`;
- использует предпочтения и риск классификаторов, если опубликованные модели предоставляют совместимые сигналы.

Риск перехода v2 может использовать сохранённую стабильность битовой сетки и структурные границы
SONARA. Это по-прежнему диагностика для порядка, проверяемого на слух: не cue points и не обещание
удачного сведения. Версия v1 сохраняет первоначальный расчёт для воспроизводимых оценок.

Последовательность SET использует структуру отдельно от компонентов риска Hybrid. При наличии
структурных данных уверенность перехода сочетает `50%` темпа, `30%` гармонической совместимости и
`20%` структуры. Без структуры используются `60%` темпа и `40%` гармонической совместимости.
Стабильность битовой сетки влияет через надёжность темпа.

Точные правила темпа, Camelot и структуры см. в
[контракте SONARA v0.2.9](../reference/sonara-v0-2-9-contract.md).

Интерфейс записывает сессии и события оценки для обратной связи. Сам черновик не изменяет треки и
текущий сет.

## Добавьте черновик

Нажмите **Add preview**, чтобы дописать треки текущего черновика SET в текущий сет. Уже добавленные
треки не дублируются. Экспорт остаётся отдельным шагом.
