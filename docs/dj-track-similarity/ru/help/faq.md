# Частые вопросы

> Для кого: Для тех, кому нужны короткие ответы о текущей исполняемой части v7.
> Задача: Отделить безопасные локальные сценарии от устаревших предположений.
> Тип: Помощь

## Изменяет ли анализ аудиофайлы?

Нет. SONARA, MAEST, MERT, MuQ, CLAP и расчёт классификаторов записывают только данные SQLite.

## Какие действия могут изменить аудиофайлы?

Только применение жанровых тегов MAEST, Audio Doctor apply и Audio Dedup apply. Audio Dedup apply
может удалить файл; каждое действие требует явного подтверждения и отделено от обычного
сканирования, поиска и анализа.

## Может ли v7 мигрировать старую базу?

Нет. Исполняемая часть Python использует новую схему v7. Она не мигрирует файлы v5/v6, не адаптирует
старые результаты SONARA и не создаёт sidecar Timeline/Representations. Фонотека состоит из Core и
обязательного `*.artifacts.sqlite`, связанных `catalog_uuid`; `*.evaluation.sqlite` хранит
необязательные данные оценки.

## Что делать после изменения SONARA?

Запустите `prepare-sonara-release` с проверенным местом для резервных копий, затем повторно
проанализируйте SONARA. Операция использует точные контракты `core`, `timeline`, `embedding` и
`fingerprint`, ведёт долговечный receipt для продолжения после сбоя и является упорядоченной, а не
распределённой атомарной. После неё переобучите, опубликуйте и пересчитайте все классификаторы,
зависящие от SONARA. Текущая ревизия признаков проекта — `6`.

## Почему артефакт классификатора заблокирован?

Для расчёта нужен манифест классификатора версии `2`. Добавленные в репозиторий артефакты версии `1`
или без версии надо переобучить и опубликовать; их оценки не используются молча.

## Можно ли использовать браузерный UI с v7?

Пока нет. Порт frontend на v7 отложен. Для работы с фонотекой v7 используйте CLI или API.

## Можно ли делиться отчётами или базами?

Сначала проверьте их содержимое: там могут быть локальные пути, теги, оценки и решения при прослушивании.
