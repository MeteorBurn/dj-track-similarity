# Обслуживайте фонотеку безопасно

> Для кого: Для тех, кто поддерживает локальную фонотеку и базу SQLite.
> Задача: Дать регулярный порядок действий без случайного изменения аудио.
> Тип: Сценарий

## Регулярные проверки

1. Запускайте **Scan** после добавления файлов. Сканирование обновит SQLite по изменившимся сведениям о файлах и прочитает теги.
2. Запускайте **Refresh Tags** после редактирования тегов внешней программой. Операция перечитает выбранные поля Mutagen для существующих треков.
3. Анализируйте только отсутствующие данные нужных моделей и выходных блоков SONARA.
4. Используйте выборочный сброс, когда намеренно хотите пересчитать одну модель.
5. Экспортируйте проверяемые подборки и сеты, не редактируя аудиофайлы.

## Перенос фонотеки

Сначала выполните предварительную проверку. Применяйте перенос только при отсутствии конфликтов и
пропавших целевых файлов. Применение изменяет лишь сохранённые пути SQLite.

Предварительная проверка в CLI:

```powershell
dj-sim relocate-library D:\Music E:\Music --db .\data\library.sqlite
```

Применение:

```powershell
dj-sim relocate-library D:\Music E:\Music --apply --db .\data\library.sqlite
```

## Сначала отчёт, потом исправление

Перед ремонтом запустите Audio Doctor в режиме предварительной проверки. Перед удалением запустите
Audio Dedup в режиме отчёта. До любого применения внимательно просмотрите отчёты XLSX.

## Обслуживание базы данных

Оптимизируйте только локальную базу SQLite, которую контролируете. Для фонотеки скрипт проверяет
Core, Timeline и Representations и создаёт их резервные копии, а затем проверяет целостность всех трёх баз
до и после обслуживания.

```powershell
python scripts\optimize_database.py --db .\data\library.sqlite
```

## Не используйте применение как тест

Режимы применения Audio Doctor и Audio Dedup действительно изменяют файлы. Не запускайте их для
обычной проверки.
