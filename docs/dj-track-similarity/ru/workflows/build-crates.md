# Собирайте подборки для последующего прослушивания

> Для кого: Для тех, кто собирает пул кандидатов, а не окончательно упорядоченный сет.
> Задача: Создать полезные списки из фильтров, опорных треков и текстовых запросов.
> Тип: Сценарий

Подборка нужна, когда хочется сначала собрать варианты и только позже определить порядок. В ней
может находиться семейство близкого звучания, направление разогрева, трудно называемые фактуры или
треки, которым стоит дать ещё один шанс.

Результат — проверенный пул шире готового сета. К нему можно вернуться позже, воспроизвести вне
приложения или использовать как исходный материал для следующего отбора.

## Порядок действий

1. Просканируйте и проанализируйте достаточную часть библиотеки для нужного вида поиска.
2. Сузьте список по исполнителю, названию, альбому, пути, жанрам, отметкам «нравится», синкопированному ритму или классификаторам.
3. Если фильтр уже точно описывает подборку, добавьте все видимые отфильтрованные треки в текущий сет.
4. Расширьте найденное звучание через MERT или SONARA от нескольких опорных треков.
5. Используйте текстовый поиск CLAP, если подборку легче описать словами, чем ориентиром.
6. Прослушайте результаты и удалите очевидные промахи.
7. Экспортируйте CSV для проверки или M3U для воспроизведения.

## Как разделить результаты

- CSV подходит для проверки метаданных и путей.
- M3U подходит для загрузки списка в проигрыватель.
- Коллекция Rhythm Lab подходит, если тот же список станет материалом для проверки классификатора.

## Приватность

Экспорт подборки содержит локальные пути и, возможно, ваши стилевые решения. Не публикуйте его без
проверки.
