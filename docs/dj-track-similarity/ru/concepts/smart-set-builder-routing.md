# Как Smart Set Builder строит маршрут

> Для кого: Для тех, кто настраивает SET.
> Задача: Объяснить требования к анализу и причины порядка в предварительном списке.
> Тип: Понятие

Smart Set Builder начинает с подходящих треков. Опорные точки и оценки кандидатов поступают в
построитель маршрута. Результат — предварительный список для прослушивания, а не автоматически
готовый сет.

## Условия участия

Треку нужны данные SONARA, MERT, MAEST и CLAP. В ответе указаны общее количество треков и объём
недостающих данных, чтобы было понятно, готова ли фонотека.

## Опорные точки вручную и автоматически

В ручном режиме выбранные референсные треки (seed) становятся фиксированными опорными точками
маршрута. В автоматическом режиме первая опорная точка выбирается из фонотеки с полным набором
признаков, а затем подбираются связанные опорные точки с учётом разнообразия исполнителей.

Бэкенд ограничивает число автоматически выбранных опорных точек диапазоном `1..5` и требует
достаточного числа подходящих треков.

## Оценка кандидатов

Базовая оценка сочетает:

- сходство эмбеддингов MERT;
- сходство аудиоэмбеддингов CLAP;
- сходство эмбеддингов MAEST;
- широкое сходство SONARA;
- необязательное предпочтение классификатора и уверенность его оценки.

Режимы смещают баланс между близким сходством, открытием нового, странными соседними совпадениями и
плавным развитием.

## Упорядочивание

При построении последовательности добавляются:

- уверенность в переходе;
- соответствие кривой энергии;
- соответствие кривой BPM, если она включена;
- давление разнообразия;
- траектория оценки классификатора;
- давление повторов исполнителя.

Для известных исполнителей действует ограничение: в одном предварительном списке может быть не более
одного трека каждого известного исполнителя. Если выбранные вручную референсные треки нарушают это
правило, SET сообщает об ошибке, а не скрывает конфликт.

## Поведение BPM

Режим General BPM использует темп для совместимости перехода. Low-to-high и high-to-low добавляют
реальную траекторию BPM. Сопоставление половинного и двойного темпа помогает оценить переход, но не
заменяет фактическое движение.

SET сначала определяет темп по актуальным данным SONARA. При уверенности ниже `0.45` рабочую оценку
могут подтвердить или заменить ранжированные варианты темпа SONARA и тег BPM из Mutagen.
Стабильность сетки долей ослабляет ненадёжные данные о темпе, а низкая надёжность сама по себе не
может стать жёстким основанием для исключения. См.
[контракт SONARA v0.3.1](../reference/sonara-v0-3-1-contract.md).

## Добавление всегда явное

SET возвращает строки предварительного списка и не заменяет текущий сет. Нажмите **Add preview**,
чтобы добавить эти треки в сет.
