# Просматривайте фонотеку, не загружая её целиком

> Для кого: Для тех, кто после сканирования работает в основном веб-интерфейсе.
> Задача: Находить треки, изучать метаданные, слушать аудио и собирать референсные треки (seed) или временный сет.
> Тип: Руководство

::: warning Состояние интерфейса v7
Описанный ниже сценарий React относится к отложенному интерфейсу. Он ещё не перенесён на API схемы
v7, поэтому эти шаги сейчас не проверены и недоступны для v7. Используйте приведённую ниже
альтернативу через API бэкенда.
:::

## Актуальная альтернатива v7

Запустите бэкенд на `127.0.0.1`, затем обращайтесь напрямую к конечным точкам фонотеки с пагинацией:

```powershell
Invoke-RestMethod -Uri 'http://127.0.0.1:8765/api/library/summary'
Invoke-RestMethod -Uri 'http://127.0.0.1:8765/api/tracks?limit=25'
```

Полные метаданные возвращает `GET /api/tracks/{track_id}`, а аудио для прослушивания —
`GET /media/{track_id}`. Актуальные структуры ответов и параметры фильтрации описаны в
[справочнике API](../reference/api.md).

## Поведение отложенного интерфейса

Панель фонотеки читает `/api/tracks` с серверной пагинацией. Вся база SQLite не загружается в
браузер целиком.

## Основные элементы

В фонотеке доступны:

- текстовый поиск по исполнителю, названию, альбому, пути, жанрам и другим индексированным полям;
- выбор режима `like` или FTS, если он показан в интерфейсе;
- пресет синкопированного ритма при наличии соответствующих данных MAEST;
- фильтр только понравившихся треков;
- минимальные оценки классификаторов при наличии результатов опубликованных моделей;
- переключатель направления сортировки;
- переход назад, вперёд и к выбранной странице.

API ограничивает размер страницы диапазоном `1..500`. Строки списка остаются компактными, а полные
метаданные загружаются только по запросу.

## Диалог метаданных

Откройте сведения о треке, когда нужен полный набор данных. В диалоге отдельно показаны:

- файловые теги, прочитанные через Mutagen;
- признаки SONARA;
- жанры MAEST;
- оценки классификаторов.

В разделе Mutagen находятся основные сведения о файле: название, длительность, формат, размер и
путь. Если они есть, также показаны сохранённые теги жанра, BPM, тональности и комментария. Кнопка
рядом с путём копирует локальное расположение файла.

Не смешивайте источники при оценке трека. Метка MAEST и жанровый тег файла содержат разные данные.

## Предварительное прослушивание

Эндпоинт `/media/{track_id}` передаёт локальный файл напрямую, если браузер умеет его воспроизводить.
AIFF, FLAC, DSD/DSF, WMA, APE, WV, M4B/M4R, TAK, TTA и небезопасные для браузера WAV временно
перекодируются в 16-битный WAV для потоковой передачи. Затем временный файл удаляется. Исходное
аудио не переписывается.

Если файл отсутствует, предварительное прослушивание возвращает ошибку и не скрывает проблему.

## Отметки «нравится»

Кнопка «нравится» записывает локальную строку SQLite и не изменяет аудиотеги. Эти отметки можно
использовать для просмотра и фильтрации.

## Референсные треки и текущий сет

Для каждой видимой строки можно:

- добавить или убрать референсный трек;
- добавить трек в текущий сет или удалить из него;
- начать или остановить прослушивание;
- открыть метаданные;
- изменить отметку «нравится».

Референсные треки используются в MERT, SONARA, SET и предварительном просмотре Hybrid. Текущий сет —
временный плейлист в правой панели. На диск он записывается только при экспорте.

## Добавление видимых треков

Интерфейс умеет добавить в текущий сет все треки, соответствующие активному фильтру. Он запрашивает
отфильтрованный список у сервера и добавляет только те треки, которых ещё нет в сете.
