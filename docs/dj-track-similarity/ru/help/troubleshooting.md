# Решение проблем по симптомам

> Для кого: Для тех, кто исправляет локальную установку, сканирование, анализ или поиск.
> Задача: Дать практические проверки, связанные с текущим поведением.
> Тип: Помощь

## Сервер сообщает об отсутствующем FFmpeg

Бэкенд требует FFmpeg уже при запуске. Установите его и добавьте в `PATH` либо задайте:

```powershell
$env:DJ_TRACK_SIMILARITY_FFMPEG = "C:\Path\To\ffmpeg.exe"
```

Затем перезапустите `dj-sim serve`.

## Интерфейс просит выбрать базу

У API нет выбранной базы SQLite. Выберите файл `.sqlite` в интерфейсе или запустите сервер с
`--db`:

```powershell
dj-sim serve --host 127.0.0.1 --port 8765 --db .\data\library.sqlite
```

## Сканирование не находит файлов

Проверьте существование папки и поддерживаемые расширения:

```text
.aif .aiff .alac .flac .m4a .mp3 .ogg .opus .wav .wave
```

Имена, начинающиеся с `._`, пропускаются.

## В задаче анализа ноль треков

Выбранные семейства могут быть завершены для всех треков, либо вы открыли не ту базу, которую
сканировали. Сбрасывайте анализ только при намеренном пересчёте.

Для классификаторов проверьте опубликованный манифест и готовые/неготовые количества. Сначала
запустите отсутствующий этап SONARA или ML, затем отдельную задачу CLASSIFIERS.

## SONARA заблокирована после обновления

Миграция на нативное декодирование намеренно блокируется, если сохранённый результат имеет старый
контракт декодера или выполнения. Создайте совместную резервную копию основной и дополнительных баз, затем выполните
явный сброс SONARA. Старые и новые результаты никогда не адаптируются и не смешиваются.

Проверьте установленный пакет:

```powershell
python -c "import sonara; print(sonara.__version__)"
```

У Core, Timeline и Representations свои подписи. После сброса отсутствующие актуальные
дополнительные строки не делают Core недействительным и могут быть рассчитаны отдельно. Следуйте
[сценарию раздельного хранения](../workflows/reanalyze-sonara-split-storage.md).

## SONARA видна, но поиск или SET считают её отсутствующей

Быстрый флаг `has_sonara_analysis` может остаться у старой строки. Поиск, SET, публичные списки
анализа и сводка фонотеки требуют актуального контракта. Проанализируйте трек заново и проверьте
происхождение и подпись в метаданных.

## Timeline возвращает пустой объект

`GET /api/tracks/{track_id}/sonara-timeline` возвращает `{}`, если Timeline не анализировался или
его подпись устарела. Диалог метаданных показывает лишь список имён и не вызывает этот эндпоинт.
Выберите Timeline и снова запустите SONARA.

## Классификатор сообщает о несовместимой подписи SONARA

Опубликованный артефакт может быть устаревшим, обученным на другом профиле или запрашивать
необязательное значение, отсутствующее у трека. Порядок восстановления фиксирован:

```text
повторный анализ SONARA -> переобучение -> публикация -> повторный расчёт оценок
```

Отсутствующие значения не заменяются на `0.0`. Метки и обратная связь сохраняются, а артефакты
только по эмбеддингам не затрагиваются.

## Запрошена CUDA, но анализ завершается ошибкой

Проверьте среду PyTorch/CUDA с помощью `dj-sim doctor`:

```powershell
dj-sim doctor
```

Попробуйте `--device cpu` или `CPU` в интерфейсе, чтобы проверить остальную задачу.

## Текстовый поиск CLAP выключен или пуст

Сначала выполните анализ CLAP:

```powershell
dj-sim analyze --models clap --db .\data\library.sqlite
```

Если выдача пуста, снизьте порог и перепишите запрос вокруг слышимых свойств, а не метаданных или
имён исполнителей.

## Мало треков подходит для SET

SET требует полный набор SONARA, MERT, MAEST и CLAP. Запустите отсутствующие семейства и проверьте
счётчики покрытия.

## Задача жанровых тегов содержит ошибки

Применение жанра действительно пишет аудиотеги. Ошибки фиксируются для каждого трека, а партия
продолжается. Проверьте журнал и права файла. WAV обрабатывается через Mutagen WAVE/ID3 и должен
завершаться с понятной ошибкой, не пытаясь чинить RIFF внутри основного приложения.

## `/docs/` сообщает, что документация не собрана

Соберите сайт:

```powershell
cd docs\dj-track-similarity
npm run check
```

Затем перезагрузите `/docs/`.
