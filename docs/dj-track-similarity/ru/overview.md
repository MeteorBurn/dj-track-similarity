# Обзор

Эта страница объясняет, что такое `dj-track-similarity`, что он делает и какие
границы безопасности применяются к обычным рабочим процессам приложения.
Прочитайте ее первой, если решаете, подходит ли проект для вашей локальной
DJ-библиотеки.

## Обзор проекта

`dj-track-similarity` - локальный инструмент анализа музыкальной библиотеки для
подготовки DJ-сетов. Он сканирует папку с аудиофайлами, сохраняет метаданные и
результаты анализа в SQLite и помогает искать треки, которые могут хорошо
работать рядом друг с другом в сете.

Это персональный энтузиастский инструмент, а не коммерческий продукт и не
формальный benchmark для audio research. Практическая цель - упростить
просмотр, тегирование, поиск и подготовку реальной локальной коллекции к DJ
использованию.

У приложения две основные поверхности:

- Python backend и CLI, устанавливаемый как `dj-sim`.
- React/Vite frontend, который отдает FastAPI backend.

Используйте CLI для повторяемых команд, пакетной работы и быстрых проверок.
Используйте web UI, когда нужно просматривать библиотеку, слушать, сравнивать
кандидатов, настраивать поиск, экспортировать временный сет или проверять
метаданные перед действием.

## Типичный рабочий процесс

Обычный путь через приложение:

1. Выбрать или создать SQLite-базу.
2. Просканировать музыкальную папку.
3. Запустить один или несколько проходов анализа: Sonara, MAEST, MERT или CLAP.
4. Просматривать постраничную библиотеку, смотреть метаданные треков и выбирать
   seed-треки.
5. Искать через SONARA, MERT или CLAP.
6. Собрать временный сет и экспортировать его как M3U или CSV.
7. При необходимости оценить треки продвинутыми classifier profiles.
8. При необходимости записать MAEST-жанры в стандартные genre tags аудиофайлов.

Не нужно запускать все семейства анализа. Практичная отправная точка:

- Сначала выполните `scan`, чтобы библиотека появилась в SQLite.
- Запускайте Sonara, когда нужны быстрый объяснимый similarity search и
  фильтры по признакам библиотеки.
- Запускайте MAEST, когда нужны сгенерированные жанры или preset `syncopated`.
- Запускайте MERT, когда нужна audio similarity от seed-треков через embeddings.
- Запускайте CLAP, когда нужны текстовые prompts вроде "dark hypnotic techno".
- Запускайте продвинутые классификаторы только после появления данных Sonara,
  MERT и MAEST для треков, которые нужно оценить.

## Ключевые возможности

- Сканирует локальные аудиопапки и сохраняет строки треков в SQLite.
- Читает фиксированный whitelist человеко-понятных Mutagen-тегов.
- Обновляет Mutagen-теги уже индексированных треков без удаления анализа.
- Переносит сохраненные пути треков после перемещения той же папки библиотеки.
- Извлекает Sonara playlist features для ритма, громкости, тональности,
  perceptual и spectral inspection.
- Строит MERT audio embeddings для поиска похожести по seed-трекам.
- Строит CLAP audio embeddings и CLAP text vectors для text-to-audio search.
- Извлекает MAEST genre labels и confidence scores.
- Сохраняет MAEST embeddings во время genre analysis.
- Оценивает продвинутые classifier models по существующим данным SONARA, MERT
  и MAEST.
- Хранит статус анализа по трекам для `sonara`, `maest`, `mert` и `clap`.
- Хранит classifier scores отдельно от file tags и model metadata.
- Держит browser библиотеки server-side paginated для больших локальных
  коллекций.
- Загружает полные метаданные одного трека только при открытии metadata dialog.
- Отдает browser previews, запускает playback по кнопке preview и
  транскодирует AIFF/AIF во временные seekable WAV-файлы.
- Экспортирует текущий временный сет как `.m3u` или `.csv`.
- Сбрасывает семейства анализа независимо в SQLite.
- Очищает локальную базу после явного подтверждения.
- Записывает стандартные genre tags из сохраненных MAEST labels только по
  явному запросу.

## Модель безопасности

Стандартное поведение приложения - database-first и read-only для аудиофайлов.

- Сканирование читает файлы и пишет только строки SQLite.
- RefreshTags перечитывает file metadata и пишет только строки SQLite.
- Анализ Sonara, MAEST, MERT и CLAP читает аудио и пишет только результаты
  анализа в SQLite.
- Продвинутая classifier scoring читает существующие данные анализа SQLite и
  пишет только classifier rows SQLite. Он не декодирует и не изменяет аудио.
- Поиск читает SQLite и не меняет ни аудио, ни базу.
- Browser preview отдает аудио для player; AIFF/AIF previews могут
  транскодироваться через `ffmpeg` во временные seekable WAV-файлы, но исходные
  файлы не перезаписываются.
- Export пишет только playlist/export файлы.
- Analysis reset удаляет только выбранные локальные SQLite outputs анализа.
- Database clear удаляет только локальные SQLite track rows, embedding rows и
  зависимые classifier-score rows.
- Library relocation обновляет только сохраненные значения `tracks.path` в
  SQLite.

Явное исключение - workflow сохранения жанров:

- `/api/tags/genres/apply` и `/api/tags/genres/jobs` могут записывать
  стандартные audio genre tags из сохраненных MAEST labels.
- Запись жанров должна перезаписывать только поле genre.
- Существующие title, artist, album, BPM, key и другие обычные tags должны
  остаться нетронутыми.

Отдельный helper `scripts/audio_repair/repair_audio_metadata.py` - еще одно
явное исключение. Он отделен от приложения, по умолчанию работает в dry-run и
может перезаписывать repairable аудиофайлы только с `--apply`.

## Поддерживаемые аудиофайлы

Основной scanner сейчас индексирует такие расширения:

```text
.aif, .aiff, .alac, .flac, .m4a, .mp3, .ogg, .opus, .wav, .wave
```

Repair helper поддерживает более широкий diagnostic set, потому что проверяет
несоответствия контейнера и codec для большего числа форматов. См.
[Восстановление метаданных](scripts/repair-audio-metadata.md) для
этого workflow.
