# Создайте первую библиотеку

> Для кого: Для тех, кто создаёт библиотеку SQLite из папки с музыкой.
> Задача: Понять сканирование, Refresh Tags, просмотр и выбор базы данных.
> Тип: Учебная инструкция

Библиотека — это личная карта вашей музыкальной коллекции внутри приложения. Она позволяет искать
по одному каталогу, даже если файлы разбросаны по разным папкам, а теги заполнены не полностью.
Аудиофайлы остаются на своих местах.

## Что даёт сканирование

- единый список поддерживаемых аудиофайлов для просмотра;
- поиск по исполнителю, названию, альбому, жанру, пути и другим доступным тегам;
- стабильные записи треков, к которым позже привязываются результаты анализа;
- счётчики добавленных, обновлённых, неизменившихся и пропущенных файлов.

Одного сканирования недостаточно для поиска по звучанию. Сначала оно создаёт каталог, а следующий
шаг — [анализ](./first-analysis.md) — добавляет аудиоданные для поиска похожих треков, текстового
поиска и черновиков SET.

## Как хранится каталог

Одна библиотека использует три соседних файла SQLite. Выбранный файл Core хранит пути, теги,
небольшие значения SONARA, эмбеддинги MAEST/MERT/MuQ/CLAP, флаги, отметки «нравится», обратную связь
и необязательные оценки классификаторов. Временные последовательности SONARA находятся в
`*.timeline.sqlite`. Необязательные эмбеддинг и отпечаток SONARA находятся в
`*.representations.sqlite`. Приложение создаёт и связывает обе дополнительные базы автоматически.

## Выберите или создайте базу

В CLI передайте `--db`:

```powershell
dj-sim scan D:\Music --db .\data\library.sqlite
```

В интерфейсе используйте кнопку выбора базы в блоке **1. Database and analysis**. Системный диалог
позволяет выбрать существующий файл `.sqlite` или указать путь для нового.

Если приложение запущено без выбранной базы, интерфейс попросит выбрать её до сканирования, поиска
или анализа.

## Просканируйте папку

В CLI:

```powershell
dj-sim scan D:\Music --db .\data\library.sqlite
```

В интерфейсе:

1. Выберите основную базу SQLite Core. Приложение автоматически откроет соответствующие файлы Timeline и Representations.
2. Введите или выберите корневую папку с музыкой.
3. Задайте **Scan workers**. Текущее измеренное значение по умолчанию — `8`.
4. Нажмите **Load tracks into database**.

Поддерживаемые расширения:

```text
.aif .aiff .alac .flac .m4a .mp3 .ogg .opus .wav .wave
```

Файлы AppleDouble, имена которых начинаются с `._`, пропускаются.

## Что читает сканирование

Через Mutagen читается фиксированный набор полезных человеку метаданных: исполнитель, название,
альбом, жанр, год, страна, лейбл, номер в каталоге, номер трека и диска, BPM, тональность,
комментарии, ISRC, длительность, аудиоформат и данные кодека, если они доступны.

Если теги прочитать не удаётся, приложение всё равно создаёт минимальную запись, используя имя файла
без расширения как название.

## Что записывает сканирование

Сканирование изменяет только основную базу SQLite Core. Треки обновляются по пути, а их записи
перезаписываются, когда меняются размер файла или время модификации. Аудиотеги не записываются.

Задача сканирования в интерфейсе показывает прогресс, события, текущий путь, счётчики и состояние
отмены. CLI выводит количества добавленных, обновлённых, неизменившихся и пропущенных файлов.

## Refresh Tags

Используйте **Refresh Tags**, когда файлы уже есть в библиотеке и нужно лишь перечитать их теги.
Команда обновляет выбранные поля метаданных существующих треков в SQLite. Она не запускает повторно
SONARA, MAEST, MERT, MuQ или CLAP.

## Просмотр после сканирования

Список библиотеки использует серверную пагинацию. Доступны режимы поиска `like` и FTS, фильтр
понравившихся треков, фильтр синкопированного ритма и фильтры оценок классификаторов, если есть
опубликованные модели.

Открывайте диалог сведений о треке только тогда, когда нужны все метаданные. Основной список остаётся
лёгким даже для больших библиотек.
