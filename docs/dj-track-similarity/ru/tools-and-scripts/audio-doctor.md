# Audio Doctor

> Для кого: Для тех, кто диагностирует проблемы метаданных или контейнера аудио.
> Задача: Сначала выполнить безопасную проверку и применять исправления только после изучения отчёта.
> Тип: Руководство

Audio Doctor проверяет известные проблемы метаданных и контейнера и умеет исправлять только известные
безопасные состояния. Всегда начинайте с предварительной проверки.

## Источник в интерфейсе

Откройте Audio Doctor значком гаечного ключа в верхней панели.

**Selected DB** читает `tracks.path` выбранной базы SQLite. Необязательные **DB roots** ограничивают
сохранённые пути, а **File root** перед проверкой файловой системы заменяет совпавшие корни базы.

**Folder** рекурсивно сканирует выбранную папку.

## Настройки интерфейса

- **keep-id3**: `first`, `last` или `none` для обработки WAV.
- **Workers**: `1..32` для предварительной проверки. Применение всегда последовательное.
- **Limit**: необязательные первые N ожидающих файлов.
- **Reason**: необязательные фильтры причин из старого состояния или отчёта.
- **Output dir**: каталог комплекта отчётов.
- **State path**: необязательный путь JSON состояния для повторных проверок и применения.

Нажмите **Start** для предварительной проверки. До исправления изучите XLSX.

## Предварительная проверка из CLI

Папка:

```powershell
python tools\audio-doctor\audio_doctor_cli.py --folder D:\Music
```

Выбранная база:

```powershell
python tools\audio-doctor\audio_doctor_cli.py --db .\data\library.sqlite
```

С заменой корня:

```powershell
python tools\audio-doctor\audio_doctor_cli.py --db .\data\library.sqlite --db-root D:\OldMusic --file-root E:\Music
```

## Режим применения

Интерфейс и API требуют точного подтверждения:

```text
APPLY REPAIR
```

Также требуется состояние предыдущей проверки. В отдельном CLI применение по умолчанию создаёт
полные резервные копии файлов, если вы явно не отключили их.

Исправляются только строки со статусом, допускающим ремонт. Это не универсальный редактор тегов.

## Выходные данные

Отчёты и состояние по умолчанию находятся в `tools/audio-doctor/data/`. Считайте их личными: внутри
есть пути и диагностика файлов.
