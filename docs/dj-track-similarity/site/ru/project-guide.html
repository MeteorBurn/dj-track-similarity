Документация dj-track-similarity ​

Это индекс документации dj-track-similarity. Начинайте отсюда, если не уверены, какая страница нужна. Краткое введение остается в README репозитория; страницы ниже содержат пользовательские и разработческие справочники.

Русская локализация переводит текущую пользовательскую документацию. Английская версия остается канонической для регулярных обновлений UI-документации; русская версия обновляется по отдельному запросу.

Для первого локального запуска начните с раздела Установка, прочитайте Обзор, затем используйте CLI, чтобы просканировать папку и запустить сервер. Если база библиотеки уже есть, откройте Анализ, чтобы выбрать следующий проход анализа, а затем Поиск и теги для ежедневного DJ-рабочего процесса.

Основной проект ​

  • Установка: требования, группы зависимостей, проверенный Windows CUDA-стек, настройка FFmpeg и проверка установки.
  • Обзор: назначение проекта, ключевые возможности, модель безопасности и поддерживаемые аудиофайлы.
  • Архитектура: карта backend/frontend, зависимости, заметки по CUDA/FFmpeg и логирование. Используйте, когда неясны установка, GPU, декодирование аудио или запуск сервера.
  • База данных: схема SQLite, метаданные, embeddings и хранилище оценок классификаторов. Используйте, чтобы понять, что приложение сохраняет и что остается только для чтения.
  • Модели: что такое каждая модель (Sonara, MAEST, MERT, CLAP), что она выдаёт, как её использует приложение и какие группы зависимостей ей нужны.
  • Анализ: Sonara, MAEST, MERT, CLAP и оценка продвинутыми классификаторами. Используйте перед долгими задачами анализа, чтобы выбрать нужное семейство признаков.
  • Поиск и теги: рабочие процессы SONARA/MERT/CLAP/CLASS и запись стандартных жанров. Используйте для поиска совместимых треков, фильтрации по классификаторам и решения, когда безопасно записывать жанры.
  • CLI: команды dj-sim, параметры, вывод и примеры.
  • Web API: FastAPI endpoints, которые использует frontend.
  • Разработка: команды настройки и рекомендации по фокусной проверке.
  • Идеи: крупные еще не реализованные рабочие процессы, включая экспериментальный set-building и active-learning концепции.

Rhythm Lab ​

  • Rhythm Lab: профили классификаторов, UI разметки, обучение, предсказания, promotion, удаление и интеграция с основным приложением.

Служебные скрипты ​

Здесь документируются только стабильные пользовательские помощники:

  • Остальные Python-файлы в scripts/ являются разовыми диагностиками, миграциями или экспериментами. Для них источником истины остаются локальный вывод --help и фокусные тесты, а не публичный maintenance guide.

  • Восстановление метаданных: диагностика и восстановление метаданных/контейнеров с dry-run по умолчанию.

  • Дубли и очистка: по умолчанию только отчётный анализ кандидатов на дубли, плюс явный подтверждаемый проход очистки --apply.

  • Оптимизация БД: проверенная по схеме резервная копия SQLite, VACUUM, ANALYZE и проверка целостности.
