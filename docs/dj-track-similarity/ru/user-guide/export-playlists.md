# Экспортируйте черновик плейлиста

> Для кого: Для тех, кто собрал временный сет и хочет получить файл вне приложения.
> Задача: Экспортировать M3U или CSV и понимать, что именно записывается.
> Тип: Руководство

Текущий сет остаётся временным до экспорта. Результаты поиска и черновики SET изменяют его только
после явного добавления треков.

## Действия в интерфейсе

1. Добавьте треки в текущий сет из библиотеки, результатов поиска, черновика SET или другой области интерфейса.
2. Укажите название плейлиста.
3. Выберите или введите выходную папку.
4. Нажмите кнопку экспорта M3U или CSV.

Если выходной папки нет, приложение создаст её.

## Имена файлов

Экспортёр преобразует название плейлиста в безопасное имя: сохраняет буквы, цифры, `.`, `_` и `-`,
заменяет остальные последовательности символов на `_`, а после пустого результата использует
`playlist`.

## Формат M3U

Файл M3U содержит:

- строку `#EXTM3U`;
- по одной строке `#EXTINF:-1,Artist - Title` для каждого трека с исполнителем и названием;
- сохранённый локальный путь к каждому треку.

## Формат CSV

Файл CSV содержит столбцы:

```text
artist,title,bpm,key,energy,path
```

## Сохранение в коллекцию Rhythm Lab

Текущий сет также можно сохранить как коллекцию Rhythm Lab. Эта операция записывает базу меток
Rhythm Lab, а не исходные аудиофайлы.

## Приватность

Экспорт содержит локальные пути и может раскрывать структуру коллекции. Считайте M3U и CSV личными
файлами, если вы намеренно не удалили из них приватные сведения.
