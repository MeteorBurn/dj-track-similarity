# Быстрый старт: сканирование, анализ и поиск

> Для кого: Для новых пользователей, которым нужен один рабочий проход до чтения подробностей.
> Задача: Создать базу, открыть интерфейс и выполнить первый полезный поиск.
> Тип: Учебная инструкция

Команды ниже создадут локальный каталог и проанализируют 25 треков, чтобы в браузерном интерфейсе
появились настоящие результаты поиска. Не увеличивайте первую партию, пока не проверите установку и
не сравните разные способы поиска.

В конце выберите знакомый трек и прослушайте кандидатов, которые приложение поставит рядом. Именно
этот первый короткий список — полезный результат быстрого старта.

Команды предполагают, что окружение Python уже активировано.

## 1. Установите базовый пакет

```powershell
python -m pip install -e ".[dev]"
```

Базовой установки достаточно для сканирования, просмотра, запуска интерфейса, экспорта, операций с
базой и работы с уже сохранёнными результатами анализа. Для новых задач анализа позже установите
дополнительные компоненты:

```powershell
python -m pip install -e ".[sonara,ml,dev]"
```

## 2. Создайте базу и просканируйте папку

```powershell
mkdir data
dj-sim scan D:\Music --db .\data\library.sqlite
```

Сканирование рекурсивно обходит папку и через Mutagen читает поддерживаемые типы аудиофайлов:

```text
.aif .aiff .alac .flac .m4a .mp3 .ogg .opus .wav .wave
```

Команда создаёт локальный каталог путей и тегов. После этого фонотеку уже можно просматривать, но
приложение ещё не умеет сравнивать звучание треков. Сканирование записывает строки SQLite и не
изменяет аудиофайлы.

## 3. Проанализируйте первую партию

Если дополнительные компоненты анализа ещё не установлены, установите их. Затем запустите небольшую
партию:

```powershell
dj-sim analyze --models sonara --limit 25 --db .\data\library.sqlite
dj-sim analyze --models maest,mert,muq,clap --limit 25 --db .\data\library.sqlite
```

Небольшой лимит позволяет проверить весь набор моделей до анализа коллекции целиком. Заодно можно
услышать, чем отличаются результаты разных способов поиска, прежде чем тратить время на полный
прогон.

- SONARA включает поиск по понятным аудиопризнакам и данные о переходах.
- MERT даёт прямой поиск «найти треки рядом с этим треком».
- MAEST и CLAP дополняют набор данных, необходимый для SET.
- CLAP также включает текстовый поиск.
- В этом выпуске MuQ используется только в **LAB Reference Compare**. Его можно не запускать, если
  нужны лишь поиск по референсному треку (seed), SET, Hybrid или текстовый поиск.

В CLI не указывайте `--limit`, чтобы обработать всю фонотеку. В интерфейсе значение
`Analyze limit = 0` означает всю фонотеку.

## 4. Запустите интерфейс

```powershell
dj-sim serve --host 127.0.0.1 --port 8765 --db .\data\library.sqlite
```

Откройте:

```text
http://127.0.0.1:8765/
```

На Windows можно воспользоваться скриптом запуска:

```powershell
run_server.cmd local --db .\data\library.sqlite
```

Команда `run_server.cmd lan --db .\data\library.sqlite` нужна только тогда, когда сервер должен быть
доступен другим устройствам в локальной сети. Пока сервер работает, его окно терминала занято.
Следующие команды CLI запускайте во втором активированном терминале либо используйте элементы
анализа в интерфейсе.

## 5. Пройдите первый сценарий в интерфейсе

1. В блоке **Database and analysis** проверьте путь к SQLite и корневую папку музыки.
2. В разделе **Library** найдите трек через поиск или страницы списка.
3. Добавьте от одного до пяти референсных треков.
4. Откройте **MERT**, чтобы увидеть широкое звуковое окружение, или **SONARA**, чтобы направлять поиск
   по ритму, тембру, динамике, гармонии и темпу.
5. Откройте **SET** и создайте упорядоченный черновик.
6. Прослушайте кандидатов, прежде чем добавлять их в текущий сет.
7. Когда список станет полезным, экспортируйте его в M3U или CSV.

Теперь у вас есть рабочий цикл: начать с идеи, получить короткий список, послушать, оставить полезные
треки и экспортировать только проверенный результат.

## Если что-то не работает

- Без FFmpeg сервер не запустится, а декодирование через FFmpeg будет недоступно. См. [Решение проблем](../help/troubleshooting.md).
- Пустой поиск обычно означает, что нужное семейство анализа ещё не запускалось. См. [Первый анализ](./first-analysis.md).
- Текстовый поиск CLAP требует заранее сохранённых аудиоэмбеддингов CLAP. Сначала выполните анализ CLAP.
