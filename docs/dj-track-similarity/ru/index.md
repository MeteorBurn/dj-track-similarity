Приложение не загружает вашу коллекцию в интернет и не решает за вас, какое сведение будет хорошим.
Оно помогает превратить большую папку с музыкой в короткий список треков, которые стоит послушать
вместе.

Бэкенд Python и CLI используют новую схему v7: Core, обязательную базу Artifacts и необязательную
Evaluation. Старые схемы не мигрируют. Перенос React отложен, поэтому для актуального контракта v7
используйте CLI или прямой API.

## Идея проекта

Проект вырос из личной задачи: в большой локальной коллекции легко потерять именно тот трек, который
сейчас нужен, даже если он уже лежит на диске. `dj-track-similarity` помогает заново открывать такую
музыку с помощью тегов, аудиопризнаков, эмбеддингов, текстовых запросов и необязательных персональных
классификаторов.

Более широкая цель — драматургия диджейского сета. Сет может быть не просто последовательностью
технически совместимых треков, а историей со вступлением, напряжением, разрядкой, главами и финальной
точкой. Приложение предлагает кандидатов для прослушивания, но последнее музыкальное решение всегда
остаётся за диджеем.

Это прежде всего личный любительский проект. Автор не претендует на экспертность в машинном обучении
или поиске музыкальной информации, а результаты моделей здесь следует воспринимать как локальные
данные для ранжирования, а не как истину. [Подробнее об идее проекта](./concepts/project-idea.md).

## Что читает приложение

- Пути к аудиофайлам и сведения о файлах.
- Доступные через Mutagen теги: исполнитель, название, альбом, жанр, год, BPM, тональность, лейбл,
  номер в каталоге, комментарии, ISRC, длительность и данные о формате.
- Декодированное аудио во время анализа, предварительного прослушивания или работы инструментов
  обслуживания.

## Что приложение записывает по умолчанию

Большинство сценариев записывает только данные в локальные базы SQLite и локальные отчёты.
Сканирование, **Refresh Tags**, анализ, поиск, прослушивание, сброс, очистка, предварительная проверка
переноса путей, расчёт классификаторов и экспорт не изменяют исходные аудиофайлы.

Операции, которые действительно работают с файлами, выделены отдельно:

- **MAEST genre tag apply** записывает стандартное поле жанра в аудиофайл.
- **Audio Doctor apply** исправляет только ранее обнаруженные исправимые файлы, после
  предварительной проверки и точного подтверждения.
- **Audio Dedup apply** удаляет подтверждённые дубликаты только после точного подтверждения.

Применение переноса путей фонотеки изменяет лишь `tracks.file_path` в SQLite и не перемещает файлы.
