Документация dj-track-similarity ​

Это индекс документации dj-track-similarity. Начинайте отсюда, если не уверены, какая страница нужна. Краткое введение остается в README репозитория; страницы ниже содержат пользовательские и разработческие справочники.

Русская локализация переводит текущую пользовательскую документацию. Английская версия остается канонической для регулярных обновлений UI-документации; русская версия обновляется по отдельному запросу.

Для первого локального запуска прочитайте Обзор, затем используйте Справочник CLI, чтобы просканировать папку и запустить сервер. Если база библиотеки уже есть, откройте Семейства анализа, чтобы выбрать следующий проход анализа, а затем Поиск и запись тегов для ежедневного DJ-рабочего процесса.

Основной проект ​

  • Обзор: назначение проекта, ключевые возможности, модель безопасности и поддерживаемые аудиофайлы.
  • Архитектура и среда выполнения: карта backend/frontend, зависимости, заметки по CUDA/FFmpeg и логирование. Используйте, когда неясны установка, GPU, декодирование аудио или запуск сервера.
  • База данных и сохраняемые данные: схема SQLite, метаданные, embeddings и хранилище оценок классификаторов. Используйте, чтобы понять, что приложение сохраняет и что остается только для чтения.
  • Семейства анализа: Sonara, MAEST, MERT, CLAP и оценка продвинутыми классификаторами. Используйте перед долгими задачами анализа, чтобы выбрать нужное семейство признаков.
  • Поиск и запись тегов: рабочие процессы SONARA/MERT/CLAP/CLASS и запись стандартных жанров. Используйте для поиска совместимых треков, фильтрации по классификаторам и решения, когда безопасно записывать жанры.
  • Справочник CLI: команды dj-sim, параметры, вывод и примеры.
  • Справочник Web API: FastAPI endpoints, которые использует frontend.
  • Разработка и проверка: команды настройки и рекомендации по фокусной проверке.

Rhythm Lab ​

  • Rhythm Lab: профили классификаторов, UI разметки, обучение, предсказания, promotion, удаление и интеграция с основным приложением.

Служебные скрипты ​

Здесь документируются только стабильные пользовательские помощники:

  • Скрипт восстановления аудиометаданных: диагностика и восстановление метаданных/контейнеров с dry-run по умолчанию.
  • Скрипт отчета о дублях аудио: только отчетный анализ кандидатов на дубли.
  • Скрипт оптимизации базы данных: проверенная по схеме резервная копия SQLite, vacuum, analyze и integrity check.
