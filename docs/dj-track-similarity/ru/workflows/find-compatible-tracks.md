# Найдите совместимые треки вокруг референсного трека

> Для кого: Для тех, у кого есть один трек и нужны близкие кандидаты.
> Задача: Выполнить поиск по референсному треку (seed) и признакам с понятным порядком запасных действий.
> Тип: Сценарий

Здесь «совместимый» означает «его стоит прослушать рядом с референсным треком в рамках выбранного вопроса
поиска». Это не гарантия гармонической, ритмической или стилевой совместимости.

Полезный результат — управляемый список альтернатив и соседних идей. Его можно экспортировать как
подборку или использовать удачные треки как новые референсные. Если нужен порядок, поместите лучших
кандидатов в предварительный список SET.

## Порядок действий

1. Найдите референсный трек в фонотеке.
2. Добавьте его в качестве референсного трека.
3. Запустите **MERT search** для кандидатов, близких в пространстве эмбеддингов.
4. Запустите **SONARA search**, если нужно точнее управлять ритмом, тембром, динамикой, гармонической окраской или темпом.
5. Снизьте порог сходства, если список слишком узок.
6. Прослушайте кандидатов до добавления в текущий сет.
7. Добавьте удачные треки в сет либо сохраните их как подборку или экспорт.

## Если одного референсного трека недостаточно

Добавьте второй или третий референсный трек, который выражает желаемое направление. Не добавляйте
несвязанные референсные треки только ради большего числа результатов: цель станет менее понятной.

## Замечание о темпе

Вес темпа SONARA использует совместимость с учётом половинных и двойных отношений. Траектория BPM в
SET — отдельная настройка: включайте её только тогда, когда фактический порядок должен двигаться к
целевому BPM.

Поиск и SET сначала используют актуальные данные SONARA. При уверенности ниже `0.45` они проверяют
ранжированные варианты темпа SONARA и тег BPM из Mutagen. Стабильность сетки долей может ослабить этот сигнал,
а низкая надёжность сдвигает оценку к нейтральной, не создавая бонуса или автоматического
исключения.

## Результат

Пока сравниваете треки на слух, держите небольшой список в текущем сете. Для повторной проверки
экспортируйте CSV, а для проигрывателя или DJ-приложения — M3U. На целевой машине убедитесь, что
пути работают.
