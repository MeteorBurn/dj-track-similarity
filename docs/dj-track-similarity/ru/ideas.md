# Идеи

Эта страница хранит крупные идеи для рабочих процессов, которые еще не
реализованы. Это продуктовые заметки, а не описание текущего поведения или
стабильных API-контрактов.

## Smart Set Builder по seed-трекам

Добавить отдельный UI-режим, который генерирует вариативный список кандидатов
для DJ-сета на основе 3-10 выбранных seed-треков. Цель не просто найти "еще
похожие треки", а сделать слой куратора поверх существующих моделей, который
выдает полезную последовательность для прослушивания, digging или подготовки
сета.

Первую версию можно сделать без обучения новой модели:

- использовать MERT для музыкальной и аудиальной близости;
- использовать MAEST как подсказку по жанру и текстуре;
- использовать SONARA для BPM, энергии и ритмической совместимости;
- использовать promoted classifier scores, например `abstract_edge`,
  `break_energy` или `voice_presence`, как сигналы предпочтения и исключения;
- исключать уже размеченные, лайкнутые, просмотренные или недавно показанные
  треки, если пользователь хочет discovery, а не подтверждение уже найденного.

Возможные режимы:

- `similar_crate`: держаться близко к seed-трекам и ранжировать кандидатов по
  сильному совпадению MERT, MAEST, SONARA и classifier scores.
- `weird_adjacent`: сохранять достаточную похожесть, но предпочитать треки с
  необычным расхождением моделей или более экспериментальной текстурой.
- `balanced_set`: генерировать DJ-ориентированный список с bridge-треками,
  совместимостью BPM/энергии и меньшим повторением слишком похожих треков
  подряд.
- `discovery`: допускать более рискованные кандидаты и резервировать часть
  списка под low-confidence треки, которые могут найти новые удачные positives
  для классификаторов.

Полезные контролы:

- выбор seed-треков;
- длина списка;
- уровень diversity;
- energy curve: warmup, balanced, peak или wave;
- classifier targets и avoid rules;
- исключение лайкнутых, размеченных, просмотренных или уже экспортированных
  треков;
- опциональные ограничения по BPM/key compatibility.

Результат должен объяснять, почему каждый трек попал в список, например
`similar_to_seed`, `bridge`, `weird_adjacent` или `classifier_match`, и
показывать базовые MERT/MAEST/SONARA/classifier scores для ручной проверки.

Возможная форма API:

```text
POST /api/set-builder/generate
```

Поля входа:

- `seed_track_ids`
- `limit`
- `mode`
- `diversity`
- `energy_curve`
- `classifier_targets`
- `exclude_reviewed`

Поля выхода:

- ранжированные track rows;
- причина выбора;
- similarity и diversity scores;
- релевантные classifier scores;
- опциональные данные порядка сета, например energy step или bridge target.

Дальше можно добавить отправку выбранных треков в Rhythm Lab, отметку
просмотренных кандидатов, экспорт M3U/XLSX или сохранение сессий для
итеративного active-learning workflow.
