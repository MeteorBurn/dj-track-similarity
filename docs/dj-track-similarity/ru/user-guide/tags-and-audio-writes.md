# Знайте, когда приложение может изменить аудиофайл

> Для кого: Для тех, кто хочет избежать случайного редактирования исходных файлов.
> Задача: Отделить операции чтения от явных операций записи.
> Тип: Руководство

По отношению к исходному аудио большинство сценариев работает только на чтение. Исключения
намеренно сделаны узкими и явными.

## Операции без изменения аудиофайлов

Аудиофайлы не редактируют:

- сканирование;
- **Refresh Tags**;
- анализ SONARA, MAEST, MERT, MuQ и CLAP;
- поиск MERT, SONARA, CLAP, SET и Hybrid;
- предварительное прослушивание в браузере;
- сброс анализа;
- расчёт классификаторов;
- очистка базы данных;
- предварительная проверка переноса путей;
- применение переноса путей;
- экспорт в M3U или CSV.

При этом операции могут записывать строки SQLite, журналы, отчёты, временные WAV для прослушивания,
созданные вспомогательные файлы или экспорт.

## Применение жанрового тега MAEST

Кнопка сохранения жанра запускает задачу записи тегов для всех треков с сохранёнными жанрами MAEST.
API отклоняет запись жанра для отдельного трека. Текущее поведение охватывает все доступные строки
жанров MAEST.

Задача записывает только стандартное поле жанра:

| Формат | Поле тега |
| --- | --- |
| MP3 | ID3 `TCON` |
| WAV/WAVE | обработка жанра Mutagen WAVE/ID3 с последующим чтением как `TCON` |
| AIFF | поле жанра ID3 |
| FLAC | `GENRE` |
| M4A/MP4/ALAC | `©gen` |

Обычные теги — название, исполнитель, альбом, BPM, тональность и остальные поля — сохраняются.
После успешной записи приложение заново читает метаданные этого трека в SQLite.

Ошибка фиксируется отдельно для каждого трека и не останавливает всю партию.

## Применение Audio Doctor

Audio Doctor сначала выполняет предварительную проверку. Режим применения требует точного текста
`APPLY REPAIR` и сохранённого результата проверки, после чего исправляет только файлы, признанные
пригодными для восстановления. Отдельный инструмент обрабатывает их последовательно и по умолчанию создаёт резервные
копии.

## Применение Audio Dedup

По умолчанию Audio Dedup только сообщает о возможных дубликатах. Режим применения требует точного
текста `APPLY DELETE`. После подтверждения он удаляет лишь безопасных кандидатов внутри выбранной
корневой папки и убирает из SQLite только записи тех треков, чьи файлы действительно удалены.

## Применение переноса путей фонотеки

Перенос изменяет только сохранённые пути в SQLite. До применения проверяются конфликты и наличие
целевых файлов. Операция не перемещает и не копирует аудиофайлы, не удаляет их и не изменяет их теги.
