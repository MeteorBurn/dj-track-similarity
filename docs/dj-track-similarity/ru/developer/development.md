# Процесс разработки

> Для кого: Для разработчиков, которые вносят локальные изменения.
> Задача: Использовать команды установки и сборки бэкенда, интерфейса и документации.
> Тип: Практическая инструкция

## Установка

```powershell
python -m pip install -e ".[sonara,ml,rhythm-lab,dev]"
python -m pytest
```

## Порты

До запуска на фиксированном порту проверьте, не работает ли уже соответствующий процесс проекта.
Основной бэкенд использует `8765`, Vite — `5173`, Rhythm Lab — `8777`.

## Сборки

Соберите интерфейс командой `npm run build` в `frontend/`. Проверьте публичную документацию командой
`npm run check` в `docs\dj-track-similarity`: она строго проверяет Vale для `README.md` и всего
дерева Markdown, затем собирает VitePress. После свежего checkout или изменения пакетов `.vale.ini`
один раз запустите `npm run vale:sync`. Для отчёта стиля без ошибки во время редактирования
используйте `npm run lint:style`. Каталог документации `site/` является создаваемым результатом и
не отслеживается Git.
