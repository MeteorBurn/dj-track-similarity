# Оптимизируйте базу SQLite

> Для кого: Пользователи, обслуживающие локальное состояние SQLite.
> Задача: Запустить обслуживание с резервной копией и проверками целостности.
> Тип: Руководство

Скрипт поддерживает актуальный каталог библиотеки из трёх файлов и отдельную базу меток Rhythm Lab.
Неизвестные или удалённые однофайловые форматы библиотеки отклоняются.

## Команда

```powershell
python scripts\optimize_database.py --db .\data\library.sqlite
```

## Поведение

Скрипт:

1. разрешает путь к базе;
2. проверяет существование файла;
3. определяет поддерживаемый тип базы;
4. для библиотеки проверяет общий идентификатор каталога и схемы Core, Timeline и Representations;
5. запускает `PRAGMA integrity_check` для каждого выбранного файла;
6. создаёт резервную копию SQLite рядом с каждым файлом;
7. обслуживает WAL, выполняет `VACUUM`, `ANALYZE`, `PRAGMA optimize` и создаёт контрольную точку WAL для каждого файла;
8. снова запускает `PRAGMA integrity_check`;
9. выводит тип, пути всех баз и копий, результаты целостности и общий размер до и после.

Имена резервных копий:

```text
library.sqlite.bak-YYYYMMDD-HHMMSS
library.timeline.sqlite.bak-YYYYMMDD-HHMMSS
library.representations.sqlite.bak-YYYYMMDD-HHMMSS
```

## Безопасность

Запускайте скрипт только для локальных баз, которые контролируете. По возможности сначала
остановите основное приложение, чтобы другой процесс не записывал данные во время обслуживания.
