# Локальная работа и безопасность

> Для кого: Для тех, кто хочет понять, каких данных касается приложение.
> Задача: Чётко разделить чтение, запись SQLite и изменение исходных файлов.
> Тип: Понятие

`dj-track-similarity` работает с локальными файлами и локальным состоянием. Обычные сценарии не
требуют загружать аудио или теги во внешний сервис.

## Локальное состояние

Приложение может создавать и обновлять:

- каталог SQLite из трёх файлов: выбранный Core `.sqlite` и соседние `.timeline.sqlite` и `.representations.sqlite`;
- журналы выполнения в `logs/`;
- экспортированные M3U и CSV;
- отчёты Audio Doctor и Audio Dedup в JSON, XLSX и журналах;
- файлы состояния Audio Doctor;
- необязательные вспомогательные индексы ANN;
- метки, предсказания, очереди, контрольные точки и артефакты Rhythm Lab;
- файлы опубликованных классификаторов.

Эти данные могут раскрывать локальные пути, теги, оценки и ваши решения при прослушивании. Не
добавляйте их в Git без осознанного решения.

## Операции, не изменяющие аудио

| Сценарий | Что записывается |
| --- | --- |
| Scan | строки треков и метаданные SQLite |
| Refresh Tags | метаданные SQLite существующих треков |
| Анализ | метаданные и флаги Core, поисковые ML-эмбеддинги, необязательные Timeline/Representations SONARA и оценки |
| Поиск и SET | обычно ничего; при обратной связи Hybrid — строки оценки |
| Предпросмотр в браузере | только временный WAV при необходимости перекодирования |
| Сброс | записи в трёх соответствующих файлах SQLite |
| Очистка базы | записи в трёх соответствующих файлах SQLite |
| Предварительная проверка переноса | ничего |
| Применение переноса | только сохранённые пути SQLite |
| Экспорт | новый файл M3U или CSV |
| Расчёт оценок классификаторов | оценки классификаторов SQLite |
| Отметка «нравится» | только состояние в SQLite |

## Операции с исходными файлами

Только три сценария могут записать или удалить исходное аудио:

1. **MAEST genre tag apply** записывает стандартное поле жанра для треков с сохранёнными жанрами MAEST.
2. **Audio Doctor apply** исправляет файлы после предварительной проверки и точного подтверждения `APPLY REPAIR`.
3. **Audio Dedup apply** удаляет безопасные дубликаты после точного подтверждения `APPLY DELETE`.

## Перенос не перемещает файлы

Применение переноса обновляет `tracks.path` в SQLite после проверки существования целей и отсутствия
конфликтов. Оно не перемещает, не копирует, не удаляет и не изменяет теги файлов.

## Адрес сервера

`dj-sim serve --host 127.0.0.1` доступен только локально. Используйте `0.0.0.0` или
`run_server.cmd lan` лишь тогда, когда намеренно разрешаете подключения с других устройств
локальной сети.
