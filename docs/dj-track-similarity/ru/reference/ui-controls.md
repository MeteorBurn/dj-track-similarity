# Элементы интерфейса

> Для кого: Для тех, кому нужны точные диапазоны и значения по умолчанию.
> Задача: Сделать настройки доступными для поиска без повторения всего руководства.
> Тип: Справочник

## База данных и сканирование

| Элемент | Диапазон или значения | Комментарий |
| --- | --- | --- |
| Database path | путь `.sqlite` | существующая или новая локальная база |
| Music root | существующая папка | источник сканирования |
| Scan workers | API `1..64`, интерфейс ограничивает примерно половиной CPU, но не более 8 | по умолчанию `8` для измеренного диска; сканирование метаданных и Refresh Tags |

## Анализ

| Элемент | Диапазон или значения | Комментарий |
| --- | --- | --- |
| Флажки моделей | SONARA, MAEST, MERT, MuQ, CLAP, CLASSIFIERS | SONARA + Core выбраны по умолчанию; SONARA, группа ML и отдельный этап CLASSIFIERS взаимоисключающие без FULL |
| FULL | включён/выключен | выбирает все модели и полный конвейер SONARA–ML–CLASSIFIERS |
| Analyze limit | `0..100000` в интерфейсе | `0` означает всю библиотеку |
| Device | AUTO, CPU, CUDA | для адаптеров MAEST/MERT/MuQ/CLAP |
| Результаты SONARA | Core, Timeline, Representations | Core по умолчанию, остальные включаются явно |
| SONARA native batch | `1..16` | одновременные полные пути для `analyze_batch`, по умолчанию `8` |
| Track batch size | `1..64` | декодированные треки на партию, измеренное значение `8` |
| Inference batch size | `1..128` | примеры за проход модели, измеренное значение RTX 3090 — `16` |
| Analyze | выбранные модели | единственная кнопка запуска, фиксированный порядок SONARA, ML, CLASSIFIERS |
| Reset | одно семейство | сброс только SQLite |

Журнал ограничивает настройки активным этапом. SONARA показывает результаты и нативную партию; ML
— модели, Device, Track batch и Inference batch; CLASSIFIERS — число профилей без параметров ML.

## Поиск

| Элемент | Диапазон или значения | Комментарий |
| --- | --- | --- |
| Similarity | `0.00..1.00` | порог MERT и SONARA |
| CLAP Similarity | порог исходной текстовой оценки | не сравнивается с MERT по опорному треку |
| Limit | `1..500` | число результатов |
| SONARA mode | balanced, vibe, sound, dj transition, custom mixer | custom включает видимый микшер и модификаторы |
| SONARA mixer | `0..5` | timbre, rhythm, dynamics, harmonic, tempo; dynamics включает диапазон громкости SONARA 2.0, harmonic — Camelot key |
| SONARA modifiers | `-1..1` | направленное смещение от контекста опорных треков, включая vocalness |
| LAB Limit | `1..100` | кандидаты Reference Compare на модель |
| LAB verdict | mood, palette, instruments, groove, genre, transition, miss | локальная обратная связь о кандидате и модели |

Темповые фильтры сначала используют актуальные данные SONARA. При уверенности ниже `0.45` они также
проверяют кандидатов SONARA и тег BPM из Mutagen. После проверки альтернатив низкая надёжность не
приводит к жёсткому исключению.

Вкладка LAB сравнивает CLAP, MERT, MuQ, MAEST и SONARA вокруг первого опорного трека. Группы остаются
раздельной диагностикой для сравнения семейств на слух.

## SET

| Элемент | Диапазон или значения | Комментарий |
| --- | --- | --- |
| Seed source | Manual, Auto | выбранные треки или автоматические ориентиры |
| Auto anchors | `1..5` | активно только в Auto |
| Set mode | similar crate, weird adjacent, balanced set, discovery | меняет баланс оценки |
| Track limit | `1..500` | опорные точки входят в лимит |
| Diversity | `0.00..1.00` | разносит кандидатов, сохраняя связь |
| Energy curve | balanced, warmup, peak, wave | давление порядка |
| BPM mode | general, low to high, high to low | траектория только вне general |
| BPM change | slow, medium, fast | активно для траектории BPM |
| Start BPM | пусто или `20..300` | автоматически, если пусто |
| Target BPM | пусто или `20..300` | автоматически, если пусто |
| Classifier Preference | `-1.00..1.00` | отсутствующие оценки нейтральны |
| Classifier Flow | flat, rise, fall | форма предпочтения в черновике |

Режимы BPM используют тот же преобразователь с учётом уверенности. `grid_stability` может ослабить
данные темпа SONARA, а ненадёжные пары движутся к нейтральному `0.5`, а не получают бонус.

## Предварительный просмотр Hybrid

| Элемент | Диапазон или значения | Комментарий |
| --- | --- | --- |
| Sources | MERT, MAEST, SONARA, CLAP | хотя бы один включён |
| Source weight | `0.00..1.00` | игнорируется для отключённого источника |
| Per-source | `1..100` | кандидаты на источник |
| Result limit | `1..100` | строки черновика |
| Risk penalty | `0.00..1.00` | диагностический штраф риска перехода |
| Use classifier preferences | включён/выключен | требует совместимые опубликованные сигналы |
