# Процесс разработки

> Для кого: Разработчики, которые вносят локальные изменения.
> Задача: Использовать команды установки и сборки серверной части, интерфейса и документации.
> Тип: Практическая инструкция

## Установка

```powershell
python -m pip install -e ".[sonara,ml,rhythm-lab,dev]"
python -m pytest
```

## Порты

До запуска на фиксированном порту проверьте, не работает ли уже соответствующий процесс проекта.
Основное приложение использует `8765`, Vite — `5173`, Rhythm Lab — `8777`.

## Сборки

В каталоге `frontend/` соберите интерфейс командой `npm run build`. В каталоге
`docs\dj-track-similarity` проверьте публичную документацию командой `npm run check`: она строго проверяет Vale для `README.md` и всего
дерева Markdown, затем собирает VitePress. После свежего клонирования репозитория или изменения
пакетов Vale один раз запустите `npm run vale:sync`; список пакетов указан в `.vale.ini`. Для отчёта стиля без ошибки во время редактирования
используйте `npm run lint:style`. Каталог документации `site/` является создаваемым результатом и
не отслеживается в Git.
