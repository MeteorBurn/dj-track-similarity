# Признаки, эмбеддинги и теги

> Для кого: Для тех, кто сравнивает вкладки поиска и панели метаданных.
> Задача: Объяснить каждый источник данных и подходящие способы его применения.
> Тип: Понятие

Интерфейс показывает несколько видов информации. Все они хранятся в SQLite, но поступают из разных
источников и помогают принимать разные решения.

## Начните с вопроса

| Вопрос | Полезные данные | Что появляется в приложении |
| --- | --- | --- |
| Что файл уже сообщает о себе? | Файловые теги | Исполнитель, название, альбом, жанр, BPM, тональность и другие метаданные с поиском |
| Какие треки находятся в одном обученном аудиоокружении? | MERT | Ранжирование по опорному треку |
| Какие треки близки по ритму, звучанию, динамике, гармонии или темпу? | SONARA | Поиск по признакам и данные о переходах |
| Какие треки подходят текстовому описанию звука? | CLAP | Результаты текстового поиска |
| Какие жанроподобные и звуковые данные добавляет другая модель? | MAEST | Видимые метки и поддержка SET и Hybrid |
| Как ещё одна модель ранжирует этот ориентир? | MuQ | Отдельная колонка LAB Reference Compare |
| Насколько трек соответствует моей размеченной идее? | Оценка классификатора | Фильтры CLASS и необязательные настройки SET или Hybrid |

Эмбеддинг — компактное представление модели для сравнения. Отдельные числа внутри него
интерпретировать не нужно. Признаки имеют прямые названия и часто объясняют, какое слышимое свойство
повлияло на результат. Файловые теги — это метаданные, прочитанные из файла или записанные через
явный сценарий тегов.

## Файловые теги

Scan и Refresh Tags используют Mutagen для чтения фиксированного набора: исполнитель, название,
альбом, жанр, год, страна, лейбл, номер в каталоге, номер трека и диска, BPM, тональность,
комментарий, ISRC, длительность, аудиоформат и сведения о кодеке, если они доступны.

Это теги исходного файла. Они могут быть неполными или непоследовательными. Приложение сохраняет в
SQLite копию, безопасную для JSON.

## Признаки SONARA

SONARA создаёт понятные аудиопризнаки и производные рабочие поля: BPM, тональность, длительность и
энергию. Они поддерживают вкладку SONARA и помогают Smart Set Builder рассуждать о ритме, динамике,
тембре, тональном содержимом, движении энергии и совместимости перехода.

Более новые результаты SONARA добавляют необязательные Camelot key, vocalness, mood,
instrumentalness, громкость, битовую сетку, структуру и тишину. Относитесь к этим оценкам как к
доступным для проверки данным анализа. На ранжирование влияют только поля, подключённые к
конкретному расчёту.

Аффинности настроения и instrumentalness показываются и сохраняются для возможных будущих
сценариев. Сейчас они не входят в сходство, SET, Hybrid или классификаторы. True peak и ReplayGain
также сохранены для возможного управления громкостью, а не прямого сходства SONARA. Скалярные
значения громкости могут входить в вариант классификатора `sonara2`. Текущее сравнение динамики
SONARA использует максимум моментальной громкости и диапазон громкости. Vocalness доступен через
явный модификатор поиска и необязательный вариант `sonara2vocal`.

Полные позиции битов и атак, метки и события аккордов, кривые темпа, энергии и громкости,
структурные сегменты и массивы сильных долей находятся в соседней базе `*.timeline.sqlite`.
Необязательные эмбеддинг и отпечаток SONARA находятся в `*.representations.sqlite`. Диалог
метаданных не загружает эти значения: в строке трека хранятся только списки имён полей, поэтому
наличие Timeline или Representations можно показать без раздувания основной базы Core. Размер
такта, уверенность в размере такта, изменчивость темпа и компактные сводки кривых остаются в
метаданных Core.
Активно используемые в поиске векторы MAEST, MERT, MuQ и CLAP находятся в одной индексированной
таблице `embeddings` базы Core.

Значения SONARA — результаты анализа, а не копии файловых тегов. Сценарии темпа сначала используют
актуальные подписанные данные SONARA. При уверенности ниже `0.45` они также проверяют ранжированных
кандидатов и тег BPM из Mutagen. Стабильность битовой сетки может ослабить надёжность; ненадёжный
сигнал движется к нейтральному и не создаёт сходство.

SONARA v0.2.9 Core хранит `bpm_confidence` рядом с исходным BPM, кандидатами темпа и тональностью
Camelot. Уверенность показывает, насколько сильно SONARA поддерживает рабочую оценку BPM.
Происхождение результата сохраняет schema 4 и установленную версию пакета, поэтому можно определить
конфигурацию анализа.

UI, CLI и API по умолчанию выбирают Core. Timeline и Representations — независимые результаты с
явным включением. Каждый идентифицируется детерминированной сигнатурой, поэтому отсутствующий или
несовпадающий дополнительный результат ставится в очередь без признания актуального Core
недействительным.

Точные границы полей и расчётов см. в
[контракте SONARA v0.2.9](../reference/sonara-v0-2-9-contract.md).

## Метки и эмбеддинг MAEST

MAEST сохраняет жанроподобные метки и аудиоэмбеддинг. Метки используются для отображения и
необязательной записи стандартного жанрового тега. Эмбеддинг может служить аудиосигналом в SET и
предварительном просмотре Hybrid.

Smart Set Builder может использовать эмбеддинг MAEST, но не превращает жанровые метки MAEST в
правила отбора.

## Эмбеддинг MERT

MERT сохраняет аудиоэмбеддинг. Вкладка MERT ищет от выбранных опорных треков в этом пространстве.
Сохранённые эмбеддинги также применяются в SET, Hybrid и Audio Dedup.

## Эмбеддинг MuQ

MuQ сохраняет отдельный аудиоэмбеддинг из аудио `float32` с частотой 24 кГц. Это самостоятельное
семейство анализа, которое можно сбрасывать независимо. LAB Reference Compare показывает соседей
MuQ для одного ориентира. MuQ не используется в поиске MERT/SONARA, SET, Hybrid, Audio Dedup или
классификаторах.

## Аудиоэмбеддинг CLAP

Анализ CLAP сохраняет аудиоэмбеддинги. Во время поиска вкладка CLAP преобразует текстовый запрос и
сравнивает его с этими векторами. SET, Hybrid и Audio Dedup используют те же аудиоэмбеддинги как
сигнал аудио–аудио, который отличается от оценки текстового запроса CLAP.

## Оценки классификаторов

Опубликованные классификаторы Rhythm Lab записывают баллы под `classifier_key`. Они необязательны.
Отсутствующие оценки остаются нейтральными в модификаторах SET и Hybrid.

## Почему источники нужно разделять

Жанровый тег файла, жанровая метка MAEST, оценка текстового поиска CLAP и сходство содержимого Audio
Dedup отвечают на разные вопросы. Используйте их вместе, но не считайте одной общей шкалой.
