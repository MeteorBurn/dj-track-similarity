# Build crates

Audience: DJs and music collectors  
Goal: make review lists without pretending the algorithm has final taste  
Type: how-to

A crate is a working list. It can be a warmup pool, a peak-time pool, a
left-field folder, or a short list to audition later.

## Recommended loop

1. Start with a narrow seed: a track, prompt, classifier, or filter.
2. Search with the mode that matches the seed.
3. Add candidates to the current set or export list.
4. Remove tracks that do not pass a listening check.
5. Repeat with a different seed or signal.

## Keep crates small

The most useful crates are reviewed crates. Large unreviewed exports usually
become another library to sort later.

Use `Track limit`, result limits, and focused prompts to keep the output small
enough to inspect.

## Mix signals deliberately

Good crate work often combines signals:

- MERT for near audio neighbors;
- SONARA for feature balance;
- CLAP for descriptive searches;
- CLASS for your own trained concepts;
- SET for ordered previews.

Do not treat MAEST genre labels as a crate builder unless your workflow is
specifically about writing or inspecting genre metadata.
