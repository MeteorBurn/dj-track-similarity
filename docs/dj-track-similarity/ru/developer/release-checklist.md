# Проверка перед выпуском

> Для кого: Сопровождающие, проводящие финальный аудит.
> Задача: Проверить безопасность, контракты, сборки и публичные формулировки до публикации.
> Тип: Практическая инструкция

## Список проверок

- Примеры не раскрывают личные пути, имена пользователей, реальные названия треков, секреты или расположение локальной базы.
- В документации CLI используется единая команда `dj-sim analyze`.
- Справочник API упоминает только действующие конечные точки.
- После затрагивающих изменений собраны интерфейс и документация.
- Изменённое поведение покрыто целевыми тестами.
- Отчёты, SQLite, журналы, node_modules и локальные модели не добавлены в индекс Git.
- Версия и схема SONARA, ревизия проекта, профиль по умолчанию и пример сигнатуры совпадают с `sonara_contract.py` и `sonara_features.py`.
- Семантическое изменение SONARA сопровождается проверкой инвалидации классификаторов, совместимости манифеста, миграции и целевых тестов SONARA.
- Боковое меню и локальные ссылки включают каждую новую страницу.

## Аудит безопасности

Проверьте каждый путь, записывающий аудиотеги или удаляющий аудио. Также проверьте перенос, сброс
анализа и очистку базы.
