# Руководство пользователя

> Для кого: Для тех, кто работает в браузерном интерфейсе.
> Задача: Объяснить ежедневные операции без погружения в код и подробности API.
> Тип: Оглавление руководства

Интерфейс помогает перейти от большой локальной коллекции к короткому списку, который стоит
послушать. Слева находятся база и настройки анализа, в центре — библиотека, справа — инструменты
поиска и построения сета.

## Начните с того, что у вас уже есть

| Что у вас есть | Что открыть | Что получите |
| --- | --- | --- |
| Трек, который задаёт верное направление | MERT или SONARA | Кандидатов, ранжированных вокруг опорного трека |
| Звучание, которое можно описать словами | CLAP | Короткий список для прослушивания по текстовому запросу |
| Несколько ориентиров и желаемое развитие | SET | Редактируемый упорядоченный черновик |
| Повторяющаяся личная оценка | CLASS и Rhythm Lab | Повторно используемый балл для фильтрации или мягкого влияния на результаты |
| Полезная группа треков | Текущий сет и Export | Файл M3U или CSV для другого проигрывателя или следующего этапа проверки |

Между этими областями можно переносить треки, не фиксируя окончательный плейлист. Результаты поиска
и SET остаются предварительными, пока вы явно не добавите или не экспортируете их.

## Работайте безопасно

Предварительное прослушивание и экспорт не обязывают принимать результат. Используйте приложение,
чтобы находить кандидатов, а затем слушайте их. Операции записи намеренно отделены от просмотра и
поиска.
