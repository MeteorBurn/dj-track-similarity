# Элементы интерфейса

> Для кого: Пользователи, которым нужны точные диапазоны и значения по умолчанию.
> Задача: Сделать настройки доступными для поиска без повторения всего руководства.
> Тип: Справочник

## База данных и сканирование

| Элемент | Диапазон или значения | Комментарий |
| --- | --- | --- |
| Database path | путь `.sqlite` | существующая или новая локальная база |
| Music root | существующая папка | источник сканирования |
| Scan workers | API `1..64`, интерфейс ограничивает примерно половиной CPU, но не более 8 | по умолчанию `8` для измеренного диска; сканирование метаданных и Refresh Tags |

## Анализ

| Элемент | Диапазон или значения | Комментарий |
| --- | --- | --- |
| Model checkboxes | SONARA, MAEST, MERT, MuQ, CLAP, CLASSIFIERS | SONARA + Core выбраны по умолчанию; SONARA, группу ML и отдельный этап CLASSIFIERS нельзя запускать вместе без FULL |
| FULL | включён/выключен | выбирает все модели и полный конвейер SONARA–ML–CLASSIFIERS |
| Analyze limit | `0..100000` в интерфейсе | `0` означает всю библиотеку |
| Device | AUTO, CPU, CUDA | для адаптеров MAEST/MERT/MuQ/CLAP |
| SONARA outputs | Core, Timeline, Representations | Core по умолчанию, остальные включаются явно |
| SONARA native batch | `1..16` | число путей к целым файлам, одновременно передаваемых в `analyze_batch`; по умолчанию `8` |
| Track batch size | `1..64` | декодированные треки в одном пакете задания; значение по результатам замеров `8` |
| Inference batch size | `1..128` | примеры за проход модели; значение по результатам замеров на RTX 3090 — `16` |
| Analyze | выбранные модели | единственная кнопка запуска, фиксированный порядок SONARA, ML, CLASSIFIERS |
| Reset | одно семейство | сброс только SQLite |

В журнале процесса отображаются только настройки активного этапа. Для SONARA показаны результаты и
размер нативного пакета; для ML — модели, Device, Track batch и Inference batch; для CLASSIFIERS — число профилей без параметров ML.

## Поиск

| Элемент | Диапазон или значения | Комментарий |
| --- | --- | --- |
| Similarity | `0.00..1.00` | порог MERT и SONARA |
| CLAP Similarity | порог исходной текстовой оценки | несопоставим с оценками поиска MERT по референсному треку |
| Limit | `1..500` | число результатов |
| SONARA mode | balanced, vibe, sound, dj transition, custom mixer | custom включает видимые микшер и модификаторы |
| SONARA mixer | `0..5` | timbre, rhythm, dynamics, harmonic, tempo; dynamics учитывает диапазон громкости SONARA 2.0, harmonic — тональность Camelot |
| SONARA modifiers | `-1..1` | направленное смещение относительно контекста референсного трека, включая vocalness |
| LAB Limit | `1..100` | кандидаты Reference Compare на модель |
| LAB verdict | mood, palette, instruments, groove, genre, transition, miss | локальная обратная связь о кандидате и модели |

Фильтры поиска с учётом темпа сначала используют актуальные данные SONARA. При достоверности ниже `0.45` они также
проверяют варианты темпа SONARA и BPM-тег Mutagen. После проверки альтернатив низкая надёжность не
приводит к жёсткому исключению.

Вкладка LAB сравнивает CLAP, MERT, MuQ, MAEST и SONARA относительно первого выбранного референсного трека. Группы остаются
раздельными и служат для сравнения семейств моделей на слух.

## SET

| Элемент | Диапазон или значения | Комментарий |
| --- | --- | --- |
| Seed source | Manual, Auto | выбранные треки или автоматические ориентиры |
| Auto anchors | `1..5` | активно только в Auto |
| Set mode | similar crate, weird adjacent, balanced set, discovery | меняет баланс оценки |
| Track limit | `1..500` | опорные точки входят в лимит |
| Diversity | `0.00..1.00` | распределяет кандидатов, сохраняя связь |
| Energy curve | balanced, warmup, peak, wave | влияет на порядок |
| BPM mode | general, low to high, high to low | траектория только вне general |
| BPM change | slow, medium, fast | активно для траектории BPM |
| Start BPM | пусто или `20..300` | автоматически, если пусто |
| Target BPM | пусто или `20..300` | автоматически, если пусто |
| Classifier Preference | `-1.00..1.00` | отсутствующие оценки нейтральны |
| Classifier Flow | flat, rise, fall | форма предпочтения по предварительному результату |

Режимы BPM используют тот же механизм определения темпа с учётом достоверности. `grid_stability` может ослабить
данные темпа SONARA, а ненадёжные пары движутся к нейтральному `0.5`, а не получают бонус.

## Предварительный просмотр Hybrid

| Элемент | Диапазон или значения | Комментарий |
| --- | --- | --- |
| Sources | MERT, MAEST, SONARA, CLAP | хотя бы один включён |
| Source weight | `0.00..1.00` | игнорируется для отключённого источника |
| Per-source | `1..100` | число кандидатов от каждого источника |
| Result limit | `1..100` | строки предварительного результата |
| Risk penalty | `0.00..1.00` | диагностический штраф риска перехода |
| Use classifier preferences | включён/выключен | требует совместимые результаты опубликованных классификаторов |
