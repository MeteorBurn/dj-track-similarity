# Решение проблем по симптомам

> Для кого: Для тех, кто исправляет локальную настройку или анализ v7.
> Задача: Дать проверки, связанные с текущим хранением и подготовкой релиза.
> Тип: Помощь

## Сервер сообщает, что FFmpeg не найден

Укажите путь к исполняемому файлу FFmpeg в `DJ_TRACK_SIMILARITY_FFMPEG` или добавьте FFmpeg в `PATH`,
затем перезапустите `dj-sim serve`.

## Выбранная фонотека не открывается

Для фонотеки v7 нужны Core `.sqlite` и обязательный соседний `*.artifacts.sqlite` с одинаковым
`catalog_uuid`. Не подменяйте их старыми файлами `*.timeline.sqlite` или
`*.representations.sqlite`. Эта исполняемая часть не мигрирует базы v5/v6.

## SONARA заблокирована после обновления

Запустите `prepare-sonara-release` с папкой резервных копий и точным подтверждением. Команда
подготавливает четыре результата `core`, `timeline`, `embedding` и `fingerprint`, сохраняет ход для
продолжения после сбоя и не является распределённой атомарной транзакцией. Повторно
проанализируйте SONARA, затем переобучите, опубликуйте и пересчитайте затронутые артефакты
классификаторов v2.

## Нет данных Timeline, embedding или fingerprint

Эти результаты находятся в обязательной базе Artifacts и должны быть явно запрошены после
подготовки релиза. Core включён всегда. Порт frontend v7 отложен, поэтому не судите о наличии данных
по текущим браузерным диалогам.

## Классификатор несовместим

Манифест v1 и артефакты без версии блокируются. Пересоберите и опубликуйте артефакт манифеста v2,
который соответствует текущим входам. Расчёт классификатора работает только с базой и ограничен
своим `classifier_key`.

## CUDA была запрошена, но анализ завершается ошибкой

Запустите `dj-sim doctor`, затем попробуйте `--device cpu`, чтобы отделить настройку устройства от
самого сценария анализа.
